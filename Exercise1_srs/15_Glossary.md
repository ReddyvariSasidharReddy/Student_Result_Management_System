# Glossary Terms

| **Term**                   | **Description**                                                                            |
| -------------------------- | ------------------------------------------------------------------------------------------ |
| Student                    | An individual enrolled in an academic institution who accesses results through the system. |
| Faculty                    | Teaching staff responsible for entering and verifying student marks.                       |
| Administrator              | User responsible for managing system operations, users, and security.                      |
| Result                     | Academic performance data including marks, grades, and pass/fail status.                   |
| Backlog                    | A subject in which a student has not met the minimum passing criteria.                     |
| SRS                        | Software Requirements Specification document that describes system requirements.           |
| Authentication             | Process of verifying user identity using login credentials.                                |
| Authorization              | Process of granting access permissions based on user roles.                                |
| Database                   | Centralized storage system for student and result data.                                    |
| Use Case                   | A description of how an actor interacts with the system to achieve a goal.                 |
| Actor                      | A user or system that interacts with the application.                                      |
| Functional Requirement     | A system feature that defines what the system should do.                                   |
| Non-Functional Requirement | A quality attribute such as performance, security, or usability.                           |
| Stakeholder                | Any individual or group with an interest in the project.                                   |
| Graph Analysis             | Visual representation of result trends and performance data.                               |
