# Non-Functional Requirements

| **NFR ID** | **Category**    | **Requirement Description**                                                                                                    |
| ---------- | --------------- | ------------------------------------------------------------------------------------------------------------------------------ |
| NFR-01     | Performance     | The system shall display student results within **2 seconds** under normal operating conditions.                               |
| NFR-02     | Security        | The system shall ensure **secure login**, role-based access control, and protection of academic data from unauthorized access. |
| NFR-03     | Usability       | The system shall provide a **user-friendly interface** that is easy to navigate for students, faculty, and administrators.     |
| NFR-04     | Reliability     | The system shall ensure **accurate data storage** and consistent availability without data loss during normal usage.           |
| NFR-05     | Scalability     | The system shall support an increase in the number of users and student records without performance degradation.               |
| NFR-06     | Maintainability | The system shall be easy to maintain and update with minimal effort by administrators or developers.                           |
| NFR-07     | Availability    | The system shall be available during academic working hours with minimal downtime.                                             |
| NFR-08     | Compatibility   | The system shall run on standard web browsers without requiring special hardware or software.                                  |
