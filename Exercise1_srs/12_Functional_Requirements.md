# Functional Requirements

| **FR ID** | **Functional Requirement** | **Description**                                                                                                         |
| --------- | -------------------------- | ----------------------------------------------------------------------------------------------------------------------- |
| FR-01     | User Authentication        | The system shall allow students, faculty, and administrators to log in using valid credentials to ensure secure access. |
| FR-02     | Student Management         | The system shall store and manage student details such as student ID, name, department, semester, and academic records. |
| FR-03     | Result Entry               | The system shall allow faculty members to enter and update student marks for each subject.                              |
| FR-04     | Result Calculation         | The system shall automatically calculate total marks, grades, and pass/fail status based on predefined rules.           |
| FR-05     | Result Viewing             | The system shall allow students to view their results, grades, and backlog information securely.                        |
| FR-06     | Backlog Identification     | The system shall identify students having backlogs and maintain previous and current backlog records.                   |
| FR-07     | Performance Analysis       | The system shall generate basic performance analysis and graphical representations using result data.                   |
| FR-08     | User Management            | The system shall allow administrators to manage user accounts, roles, and permissions.                                  |
| FR-09     | Report Generation          | The system shall generate result summaries and performance reports for faculty and management.                          |
| FR-10     | Data Storage               | The system shall securely store and retrieve academic data from the database.                                           |
